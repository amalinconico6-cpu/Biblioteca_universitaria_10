void it.unisa.bibliotecajfx.controller.ControllerListaUtenti.configuraColonneSePresenti ( )
private

Configura le colonne della tabella se presenti.

Esegue una configurazione basata sull'ordine delle colonne presenti nella TableView. Collega le colonne ai campi del modello Utente (nome, cognome, matricola, mail).

Postcondition
Se la tabella contiene almeno 4 colonne, vengono configurate con i rispettivi campi di Utente

◆ onRimuoviUtente()

void it.unisa.bibliotecajfx.controller.ControllerListaUtenti.onRimuoviUtente ( ActionEvent event) throws IOException
private

Rimuove l'utente selezionato e salva su file.

Chiede al modello di rimuovere l'utente. Se l'esito indica rimozione avvenuta: Salva la lista utenti su file Rimuove la riga dalla tabella Mostra popup di successo

In caso contrario mostra popup di errore.

Parameters
event
Exceptions
IOExceptionerrore durante operazioni di I/O (salvataggio)
Precondition
listaUtenti != null e deve esserci un utente selezionato
Postcondition
Se rimozione riuscita: file aggiornato e tabella aggiornata
